# 메타데이터

검색 엔진을 사용할 떄, 어떤 웹페이지가 관련이 있는지 보여주는 데 메타데이터가 사용된다.

웹사이트를 론칭할 때 큰 부분중 하나가 '검색 엔진 최적화' = SEO라고 불리는 과정이다.

웹사이트의 내용 / 의미구조 / 전달방법을 개선해 검색 결과에서의 순위를 높이는 것을 포함한다.

검색 엔진이 어떤 웹사이트를 방문하면, 

HTML 문서와 미디어 콘텐츠를 분석하고, 

다른 HTML 문서로의 링크를 발견하면 그 문서를 따라가서 모든 웹사이트를 분석할 때까지 계속 링크를 따라간다. 

분석 결과와 웹사이트의 콘텐츠에 기반하여, 검색 엔진은 다양한 검색어에 대해 웹사이트의 순위를 매긴다.

    메타 태그는 웹 페이지에 대한 메타데이터를 정의한다.

    메타데이터란 다른 데이터에 대한 데이터인데, 이 경우엔 웹 페이지에 대한 데이터이다.

    메타 태그는 html문서의 웹 페이지상에서는 보이지 않는 head 요소 안에 추가되고,

    name과 content라는 두 가지 속성을 가지고 있다.

예시 : 해리 포터 판매 사이트

저자 메타데이터

    <meta name="author" content="J.K Rowling">

    웹 페이지가 누구에 의해 작성되었는지를 알려준다.

설명 메타데이터

    <meta name="description" content="해리 포터 최신판을 지금 구매하세요!">

    웹 페이지를 설명하고, 검색 결과에서 사람들이 볼 수 있는 짧은 텍스트이다.

로봇 메타데이터

    <meta name="robots" content="index, follow">

    검색 엔진이 페이지를 분석하고 다른 페이지로의 링크도 연관해서 알려줌, noindex / nofollow로 검색 엔진이 페이지를 무시하도록 할 수도 있음.

뷰포트 메타데이터

    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    웹사이트가 핸드폰, 태블릿 등의 기기에서도 잘 보이게 해줌.

이 메타데이터들을 잘 이용하면 웹사이트가 검색 엔진에서 더 높은 순위로 뜨게끔 할 수 있다.


# SNS 메타 태그 UX

소셜 네트워크에서 링크를 공유할 때 미리보기가 생성되는 것은 메타 태그 덕분이다.

메타 태그들이 웹 페이지의 정보를 소셜 네트워크에 전달해주기 때문이다.

특히, 'Open Graph Protocol'이라는 걸 사용하면, 웹 페이지가 자신을 소셜 네트워크에 어떻게 표현할지 정의할 수 있다.
